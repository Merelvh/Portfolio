E-learning app assignment

About the project

For the project I was challenged by a company to create a workflow for an e-learning app on a topic of my choice. The app should motivate beginners to learn new skills and facilitate learning. The design and user experience should be intuitive and easy to use for complete beginners.

The workflow must consist of a set of screens (max 20 screens) encompassing a logical and intuitive navigation. It was not necessary to make an extensive workflow, but just core screens.

Styling must be interesting and eye-catching. A brand identity document wasnâ€™t necessary.

Following features should be included:

  • Task overview
  • Single lesson example
  • Single lesson exercise
  • Progress tracker

Results

Most apps show just different drawings in a step-by-step process. In the reviews users place comments about that they rather watch YouTube videos or gain more background knowledge.

Drawing apps out there provide different cartoon-ish drawing with a step-by-step process that they can execute on a paper.

Drawing & Painting provides a collection of categorized YouTube videos for different types of drawings.

Sketch a Day gives prompts for people to make an artwork and share within the app with other users.

Brief

What is the problem?

  • There isn't a proper app out there to learn how to draw following proper lessons, videos and doing tests.

What does it do?

  • Teaching people how to draw or improve their drawing skills and test their knowledge.

How does it work?

  • Users can complete different classes on different levels to gain more skills and knowledge about drawing and test their knowledge.

Why should someone use it?

  • It provides a fun and interactive tool to learn how to draw.

One line description for the app:

  • A fun interactive app that gives the knowledge and skills to learn how to draw by receiving feedback from real draw fanatics.

One unique selling point of the app:

  • Receive feedback from real draw fanatics.

What makes this app different from competitors?

  • Thanks to the feedback-feature, users are able to give and receive feedback to other users to improve their skills even more.

Target audience

The website/app will focus on the following target audiences:

Roles (groups of people with similar goals)

  • People who wan to learn how to draw
  • People that want to improve their drawing skills
  • Creative people
  • Art enthusiastics

Demographics

  • Male and female
  • Parttime and fulltime working folks
  • 20 - 50 years old

User needs

The app enables the user to:

  • Learn how to draw on any starting level, but mostly for beginners
  • Stop and continue with learning whenever they want
  • Feel motivated to continue learning

Content requirements

The user will be looking for:

  • Valuable drawing lessons
  • Explanation videos
  • knowledge about drawing
  • Interesting facts about drawing/art
  • Motivation to continue learning

Functionality requirements

The user will be able to:

  • Stop and continue with learning whenever they want
  • Gain knowledge about drawing and the basics
  • Watch drawing explanation videos
  • Complete quizzes about art/drawing
  • Select their drawing level during the onboarding process
  • Complete drawing assignments
  • Make photos from her drawings to other users to receive feedback
  • Give feedback to other users before being able to receive feedback themselves (feedback to 2 other drawings)

Wireframes

First, I created a number of empty screens and started writing down in bullet points what it should contain. The next step was to map out the functionality of screens and to create a logical placement of elements and features.

The dashboard/overview shows an overview of the course completion and motivates the user to start the next lesson or do a quiz.

The lessons would consist of an educational video, text and photos. After lessons, there would be a practical assignment where users use their knowledge to create drawings. The drawing will be reviewed by other users according to review criteria. But first the user needs to review he drawings of at least two other users.

Quizzes are included in the features to motivate the user to learn more about art. With quizzes the user can earn points for the leaderboard. Every quiz shows its progress of completion. Every completed quiz gives 10% completion. When the user makes a mistake, theyâ€™ll lose a heart for which they have to wait before they regenerate. This waiting in between makes the user want to keep playing the game.

I added a point system and leaderboard that motivates the user to continue learning to score high on the leaderboard.

The design

For the design I wanted it to portray a playful look and made use of joyful illustrations.

I choose orange for the primary color and blue (on the opposite of the color wheel) to be the secondary color. The color psychology of orange is that it portrays creativeness, positivism, optimism but also encouragement and motivation.

User testing

When I finished my clickable prototype, I asked one user to use it while I observed. To receive feedback, I asked some testing questions.

  • What is the first thing you notice on this screen?
  • What is the first thing you would do on this screen?
  • Your task is X. What would be the first thing you would do here to accomplish that task?
  • Is there anything on this screen that seems out of place?
  • Is the prototype easy to use?
  • What would you change about the prototype?
